Mnemon: Local RAG Server

A self-hosted AI inference and RAG server running entirely on local hardware. No API costs, no data leaving the network, no rate limits.

OllamaAnythingLLMDockerRAGUbuntuNVIDIA CUDA

The Idea

I wanted something that felt like ChatGPT, but for my own data, without sending anything to the cloud. I’ve spent years building systems and documenting things, but finding that information later is always harder than it should be. Search works when you know what you’re looking for. It falls apart when you don’t.

The goal was simple: take everything I care about (documents, notes, configs, articles) and make it searchable through natural language. Ask a question, get an answer, and know where it came from. No API costs, no rate limits, no data leaving the network.

I called it Mnemon, a nod to memory and recall. Fitting for a system whose whole job is to remember things so I don’t have to.

The Stack

Core Stack

  • Ollama: local LLM inference (systemd service, GPU-accelerated)
  • AnythingLLM: RAG pipeline, document ingestion, chat UI
  • Docker: container for AnythingLLM with GPU passthrough
  • LanceDB: built-in vector database (via AnythingLLM)
  • Ubuntu 22.04 LTS: host OS

Hardware

  • RTX 3070 (8GB VRAM): primary inference GPU
  • 64GB ECC RAM: memory for embeddings and model context
  • 447GB SSD: OS drive
  • 745GB SSD: dedicated AI data drive (/mnt/aidata)

Models Running

  • gemma4:e4b: primary chat model
  • mistral:7b-instruct-q4_K_M: fallback chat model
  • nomic-embed-text: embeddings for RAG retrieval

How It Works

Ollama runs as a GPU-accelerated systemd service and exposes a local API on the network. AnythingLLM runs in Docker with GPU passthrough via the NVIDIA Container Toolkit. It connects to Ollama over the LAN IP (not localhost, which doesn’t reliably resolve inside Linux containers) and handles all document ingestion, chunking, embedding, and retrieval.

Model weights and all persistent data live on the dedicated AI drive, keeping AI workloads completely isolated from the OS. The AI data drive had to be owned by the Ollama system user, not my login user, which was one of the first things I had to sort out.

What It’s Used For

  • Document Q&A: upload PDFs, notes, or articles and query them through chat
  • Home Assistant AI integration: Ollama as the local Assist backend, no cloud dependency
  • Frigate camera analysis: vision model generating descriptions for local surveillance events
  • General AI experimentation without API costs or rate limits

What I Learned

  • host.docker.internal doesn’t reliably resolve inside Docker containers on Linux. Use the server’s LAN IP instead.
  • Set OLLAMA_MODELS before pulling models or they land on the OS drive instead of the data drive.
  • Ollama runs as its own system user. The data drive permissions need to match, not your login user.
  • Most of the value in RAG comes from the data pipeline, not the model. Good context beats a bigger model every time.
